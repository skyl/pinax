.. _ref-themes:

Themes
======

A theme is a group of static media and templates that work together.
Currently there is only one theme in pinax, 'default'.
In your project, the theme is set via the :ref:`PINAX_THEME <settings-pinax-theme>` setting.

Static media for a pinax theme exist in ``pinax/media/<theme_name>/``.
Templates for a pinax theme exist in ``pinax/templates/<theme_name>/``.

The ``clone_project`` command adds settings such that
media will be found first in the ``youproject/media`` directory,
whether using runserver and the staticfiles view or using the 
``build_static`` management command to gather all of the static media.
If not found under your project, ``pinax/media/<theme_name>/`` dir is tried
thanks to the staticfiles app and the ``STATIC_DIRS`` setting::

    STATICFILES_DIRS = [
        os.path.join(PROJECT_ROOT, "media"),
        os.path.join(PINAX_ROOT, "media", PINAX_THEME),
    ]

Likewise, templates will be found if they exist in ``yourproject/templates`` or they 
will fallback to the rational defaults::

    TEMPLATE_DIRS = [
        os.path.join(PROJECT_ROOT, "templates"),
        os.path.join(PINAX_ROOT, "templates", PINAX_THEME),
    ] # and then finally to the app/template dirs by default

Thus, you can use a theme but override as much as you need to.

.. _ref-template-inheritance:

Template inheritance
--------------------

Typical inheritance for a response rendering in pinax is like that of the 
bookmarks list for example::

    base.html --> site_base.html --> bookmarks/base.html --> bookmarks/add.html

The only template that is required of your project is ``site_base.html`` which is
automatically put in place by :ref:`clone_project <customization-cloneproject>`.
``base.html``, ``bookmarks/base.html``, and ``bookmarks/add.html`` are supplied by the theme
but may be overridden by placing them in ``yourproject/templates``.
